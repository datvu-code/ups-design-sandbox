## Overview

UpS is a light-mode B2B product UI for Vietnamese e-commerce merchants managing orders, products, ads, and fulfillment. The interface is information-dense: order tables, filter bars, status tabs, bulk action toolbars, and multi-section create forms.

The layout background is `{colors.canvas}` (#f3f5f8 — a cool gray). White `{colors.surface}` (#ffffff) cards float on top with `{colors.border-secondary}` borders and a three-layer box shadow. The single chromatic accent is **UpS orange** `{colors.primary}` (#e65018) — algorithm-derived scale covers hover (#ff845c) and active (#b91800) states. No second chromatic accent.

Built on **Ant Design 6.x** with CSS variables mode (`--ant-*`). All token values are accessed via `var(--ant-*)` in custom styling. ThemeContext is the single source of truth.

**Key characteristics:**
- Light canvas, white surface panels — never dark mode
- Orange as the only brand accent — CTAs, active states, focus rings
- Roboto 14px base — dense, legible, no display type
- 8/16/24px spacing scale — no arbitrary values
- antd component system — no custom component primitives
- Tabler icons — no @ant-design/icons

## Colors

### Brand
- **Orange** (`{colors.primary}`): Primary CTA buttons, active tab underlines, focus rings, selected states. Derived scale generated by antd algorithm.
- **Orange Hover** (`{colors.primary-hover}`): Hovered primary button, hovered link.
- **Orange Active** (`{colors.primary-active}`): Pressed primary button.

### Surface
- **Canvas** (`{colors.canvas}`): Page background — `#f3f5f8`. All modules render on this.
- **Surface** (`{colors.surface}`): Card / panel background — `#ffffff`. Sections, table rows, sidebars, toolbars.
- **Border** (`{colors.border}`): Strong border — `#d5d7db`. Dividers, input outlines.
- **Border Secondary** (`{colors.border-secondary}`): Subtle border — `#edeef0`. Card edges, row separators.

### Text
- **Text** (`{colors.text}`): Primary text — `#0f1215`. Titles, labels, table cell content.
- **Text Secondary** (`{colors.text-secondary}`): Secondary — `#404246`. Descriptions, meta.
- **Text Tertiary** (`{colors.text-tertiary}`): Tertiary — `#707274`. Icons, placeholders, timestamps.
- **Text Quaternary** (`{colors.text-quaternary}`): Disabled — `#9d9ea1`.

### Semantic
- **Success** (`{colors.semantic-success}`): `#007d00` — completed status, positive indicators.
- **Warning** (`{colors.semantic-warning}`): `#a44300` — deadline warnings, pending states.
- **Error** (`{colors.semantic-error}`): `#e74850` — overdue, failed, cancelled.
- **Info** (`{colors.semantic-info}`): `#2456d3` — informational badges, links.

## Typography

Font: **Roboto** — single family, no display cut. Dense product UI — no headlines over 16px.

| Token | Size | Weight | Use |
|---|---|---|---|
| `{typography.section-title}` | 16px | 600 | Section card titles (Thông tin sản phẩm, etc.) |
| `{typography.body}` | 14px | 400 | Default — all table cells, form values, descriptions |
| `{typography.strong}` | 14px | 600 | Emphasized values — prices, totals, order IDs |
| `{typography.label}` | 14px | 400 | Form labels, column headers |
| `{typography.body-sm}` | 12px | 400 | Meta, timestamps, secondary column data |

**Rules:**
- Never use type above 16px for UI content — this is a dashboard, not a marketing page
- Right-align numbers (prices, quantities) for column comparison
- Left-align all other text
- `Typography.Text type="secondary"` for meta; `type="tertiary"` for icons and timestamps

## Layout

### Spacing System

Base unit: **8px**. Formula: `y = 8 + 8n`. Never invent intermediate values.

| Token | Value | Use |
|---|---|---|
| `{spacing.xxs}` | 4px | Icon-to-label gap, badge padding |
| `{spacing.xs}` | 8px | Inline gaps, small component internal padding |
| `{spacing.sm}` | 12px | Compact row padding |
| `{spacing.md}` | 16px | Section gap, standard row padding |
| `{spacing.lg}` | 24px | Section card padding, column gap |
| `{spacing.xl}` | 32px | Large section separation |
| `{spacing.xxl}` | 48px | Page-level vertical rhythm |

### Page Layout

- Max content width: **1440px**, centered, on `{colors.canvas}`.
- Two-column pages (create/edit forms): main content `flex: 1` + sidebar `flex: 0 0 280px`, gap 24px.
- Single-column pages (list views): full width with 24px horizontal padding.

### Section Cards

Every distinct UI region is a `{components.section-card}`:
- White background, `{rounded.lg}` 8px radius, `{colors.border-secondary}` border, tertiary box-shadow, 24px padding, 16px margin-bottom.
- Title: `{typography.section-title}` — 16px / 600 — left-aligned, 16px below it.

### Sticky Action Bar

Forms always have a sticky action bar at the bottom:
- Height ~56px, `{colors.surface}` background, `{colors.border-secondary}` top border.
- Three-tier button hierarchy (right-aligned): **text** (cancel) → **default** (save draft) → **primary** (confirm action).
- Add `paddingBottom: 64px` to all scrollable columns to prevent action bar overlap.

## Components

### Buttons — Three-Tier Hierarchy

Every form or page must have a clear primary/secondary/tertiary action distinction:

| Tier | antd type | Use |
|---|---|---|
| Primary | `type="primary"` | One per view — the main destructive or confirmation action |
| Default | `type="default"` | Secondary — save draft, export, filter apply |
| Text | `type="text"` | Tertiary — cancel, dismiss, low-importance actions |

Never use two `type="primary"` buttons in the same action group.

### Section Cards

```tsx
const sectionStyle: React.CSSProperties = {
  backgroundColor: 'var(--ant-color-bg-container)',
  borderRadius: 'var(--ant-border-radius-lg)',
  padding: 'var(--ant-padding-lg)',
  marginBottom: 'var(--ant-margin)',
}
```

Title always `Typography.Text strong` at `fontSize: 'var(--ant-font-size-lg)'`.

### Table Rows (Order List pattern)

Each row is a `{components.table-row}` card:
- White background, `{rounded.lg}` border, `{colors.border-secondary}` border, 16px margin-bottom.
- Row header: shop name + order ID + status badge, `{colors.border-secondary}` bottom border.
- Row body: flex columns with `minWidth` per column; horizontal scroll at `TABLE_MIN_WIDTH`.
- Hover: border upgrades from `{colors.border-secondary}` → `{colors.border}` via `.module.css`.

### Status Badges

Use antd `Tag` with semantic color props — never hardcode badge colors:
- `color="success"` / `color="warning"` / `color="error"` / `color="processing"` / `color="default"`

### Form Fields (Sidebar pattern)

```tsx
const labelStyle: React.CSSProperties = {
  fontSize: 'var(--ant-font-size-sm)',
  color: 'var(--ant-color-text)',
  marginBottom: 8,
  display: 'block',
}
const rowStyle: React.CSSProperties = {
  marginBottom: 'var(--ant-margin)',
}
```

Label above input, 8px gap, 16px between rows.

### Selects

Every `Select` must have `suffixIcon={<IconChevronDown size={14} />}`. No exceptions — antd's default icon is replaced project-wide with Tabler.

### Icons

- Source: `@tabler/icons-react` only. No `@ant-design/icons`.
- Size: 14px for inline/form icons, 16px for toolbar icons, 24px+ for empty states.
- Color: always `style={{ color: 'var(--ant-color-text-tertiary)' }}` — never hardcoded.

### Note Type Toggle

Use antd `Segmented` — never raw `<button>` elements:
```tsx
<Segmented
  value={noteType}
  onChange={(v) => onNoteTypeChange(v as NoteType)}
  options={[{ label: 'Nội bộ', value: 'internal' }, { label: 'Khách hàng', value: 'customer' }]}
/>
```

## Do's and Don'ts

### Do
- Use `var(--ant-*)` tokens for all custom element styling — no hardcoded hex or px values.
- Keep `ThemeContext.tsx` as the single theming file — only brand overrides that differ from antd defaults.
- Load `/principles-of-design` + `/design-principles` skills before generating any UI.
- Build one section at a time, run `/verify` after each before continuing.
- Use `Segmented` for toggle groups, antd `Tag` for badges, antd `Skeleton` for loading states.
- Add `paddingBottom: 64px` to any scrollable column that sits above a sticky action bar.
- Use `whiteSpace: 'nowrap'` on `Radio.Group` in flex rows to prevent wrapping.

### Don't
- Don't hardcode colors, spacing, or border radius — every value must come from a `var(--ant-*)` token.
- Don't use `theme.useToken()` for styling — CSS variables only.
- Don't import from `@ant-design/icons` — Tabler only.
- Don't create `.module.css` files unless pseudo-selectors (`:hover`, `:focus`) are required.
- Don't generate all components at once — section by section with live verification.
- Don't use a bad screenshot as a reference — describe intent instead.
- Don't add `type="primary"` to more than one button per action group.
- Don't use `wrap` on a flex row that contains a `Radio.Group` — the radio wraps off-screen.

## Generation Workflow

Follow this sequence for every new screen:

```
1. Load /principles-of-design + /design-principles
2. Describe intent (not a screenshot reference if reference has poor hierarchy)
3. Break screen into regions — one component per region
4. Generate one section at a time
5. /verify after each section
6. Fix before moving to next section
```

**File placement:**
- Route page → `src/app/routes/[ScreenName].tsx`
- Feature components → `src/app/modules/[Module]/components/[Screen]/`
- Mock data → `src/mock-data/[module].ts`
- Types → `src/app/modules/[Module]/types/index.ts`
